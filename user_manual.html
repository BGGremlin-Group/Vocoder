BGGG Vocoder V5.0 - User Manual

Welcome to the BGGG Vocoder V5.0 User Manual. This guide explains everything about installing, configuring, and operating your vocoder. Built with the ParrotOS neon hacker aesthetic, it’s designed for professionals, audio artists, and the BGGremlin Group community.

1. Overview

The BGGG Vocoder is a professional-grade voice manipulation suite with:

  • Few-shot voice cloning using Coqui-TTS with SpeechBrain speaker embeddings
  • Prosody and style transfer for emotional cloning
  • Real-time voice adaptation and streaming
  • Aggressive noise reduction with WebRTC VAD
  • Advanced visualization (Spectrograms, 3D Particle Swarms)
  • Facial animation with dlib landmark tracking
  • PyQt5 desktop GUI and Flask Web UI for remote control

2. System Requirements

  • Python 3.10+ (recommended)
  • OS: Linux, macOS, Windows
  • RAM: Minimum 8GB (16GB+ recommended for cloning)
  • Optional CUDA GPU for faster TTS generation

3. Installation

Clone the repo and install dependencies: 

git clone https://github.com/YourUsername/BGGG_Vocoder.git
cd BGGG_Vocoder
pip install -r requirements.txt

On Linux, you may also need: 

sudo apt install portaudio19-dev

4.1. Assets Directory Explained

  • logo_screen.png: Displayed on app startup splash screen
  • credits_screen.png: Credits/about window
  • exit_screen.png: Shown on graceful exit or shutdown
  • static_visualizer.png: Used as background in visualizers
  • shape_predictor_68_face_landmarks.dat: Dlib face model (auto-downloadable)
  • shape_predictor_68_face_landmarks.dat.fallback: Offline fallback model
  • default_model.obj: Placeholder 3D model
  • waifu_mascot.obj: Your custom mascot 3D model (e.g. anime waifu)

4.2. Themes Directory

  • web_ui.html: Fully-featured Web UI served by Flask
  • custom_theme.css: Neon ParrotOS terminal styling

4.3. Voice Models Directory

  • trained_voice_model.pt: Placeholder (replace with your trained cloning model)

4.4. Recordings Directory

  • Stores your saved .wav recordings from the vocoder output

4.5. Presets Directory

  • Contains user-defined .json presets for vocoder configuration
  • Example presets include:
    • Spooky Russian: Heavily modulated, intimidating style
    • FBI: Clean, neutral, professional testimony style
    • Witness Protection: Maximum anonymity with heavy filtering

4.6. Config Directory

  • config.json: Stores last-used settings and app state

4.7. Docs Directory

  • user_manual.html: This file. Fully hyperlinked manual for your users.

5. Running the App

Launch with: 

python main.py

On first run, it will:

  • Create missing folders automatically
  • Download required face landmark models if missing
  • Generate placeholder Web UI HTML if needed

6. GUI Features

  • Load and save presets
  • Start/stop real-time audio processing
  • Record .wav files
  • Adjust pitch shift, modulation, volume live
  • Choose vocoder algorithm: Phase, LPC, Channel
  • Enable/disable voice cloning
  • Pick visualization modes (2D Spectrogram, 3D Waterfall, Particle Swarm, etc.)

7. Web UI Features

  • Accessible via browser on localhost (default port 5000)
  • Remote configuration of vocoder parameters
  • Start/stop audio processing
  • Supports mobile or tablet control
  • Styled with ParrotOS neon color theme

8. Using Presets

Presets are simple JSON files storing vocoder settings: 

{
  "preset_name": "Spooky Russian",
  "algorithm": "channel",
  "pitch_shift": -4,
  "modulation_index": 1.0,
  "modulation_rate": 2.5,
  "volume": 0.9
}

Store them in presets/. Load them in the GUI to instantly switch styles.

9. Training Voice Models

To enable cloning, replace trained_voice_model.pt with a real checkpoint.

  • Use Coqui-TTS or SpeechBrain pipelines to train your voice.
  • Save in voice_models/.
  • App will automatically detect and use it for real-time cloning.

10. Troubleshooting

  • Missing assets? Check assets/ folder and regenerate or re-download.
  • PyAudio errors? Install system audio dependencies (e.g. portaudio).
  • Shape predictor missing? App auto-downloads, but you can manually fetch from dlib.net.
  • GPU issues? Force CPU mode by installing CPU-only torch wheels.
