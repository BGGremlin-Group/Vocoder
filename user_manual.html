BGGG Vocoder V5.0 - User Manual

Welcome to the BGGG Vocoder V5.0 User Manual. This document provides thorough instructions for installing, configuring, and using the BGGG Vocoder application, developed by the BGGremlin Group.

1. Introduction

The BGGG Vocoder is an advanced, professional-grade audio processing and voice-cloning application. Designed for audio engineers, hackers, artists, and AI enthusiasts, it features real-time processing, few-shot voice cloning, noise reduction, visualization, and interactive GUI and Web UI components—all in a hacker-themed neon aesthetic inspired by ParrotOS.

2. System Requirements

  • Python 3.10 or higher recommended
  • OS: Linux, macOS, Windows
  • RAM: 8GB minimum (16GB+ recommended for voice cloning)
  • Optional GPU for faster TTS inference (CUDA-supported)

3. Installation

Install all dependencies using the provided requirements.txt: 

pip install -r requirements.txt

Ensure you have pyaudio and audio drivers installed on your system. On Linux: 

sudo apt install portaudio19-dev

5. First Run

To start the program, run: 

python main.py

On first run, the app will:

  • Create required folders if missing
  • Download the Dlib shape predictor if not present
  • Generate default Web UI HTML if missing

6. Features Overview

  • Few-shot voice cloning with Coqui-TTS and SpeechBrain embeddings
  • Prosody and style transfer between speakers
  • Real-time voice cloning adaptation during live streaming
  • Aggressive noise reduction with WebRTC VAD
  • Interactive visualizations (Spectrogram, Mel Spectrogram, 3D Waterfall, Particle Swarm)
  • Audio-reactive facial animation with landmark detection
  • Batch processing of multiple audio files
  • Full-featured PyQt5 GUI and Web UI
  • Custom themes with ParrotOS neon hacker aesthetic

7. Configuration

The application reads settings from config/config.json and from presets in presets/. Presets define:

  • Vocoder algorithm: phase, lpc, channel
  • Pitch shift (semitones)
  • Modulation index and rate
  • Volume scaling
  • Visualization mode
  • Voice cloning model path
  • Noise reduction aggressiveness

Example preset snippet: 

{
  "algorithm": "phase",
  "pitch_shift": 0,
  "modulation_index": 0.2,
  "modulation_rate": 1.5,
  "volume": 1.0
}

8. Using the GUI

The PyQt5 GUI provides:

  • Preset loading and saving
  • Real-time microphone input and output
  • Visualization selection and customization
  • Voice cloning model selection
  • Audio recording and batch processing

9. Using the Web UI

The Flask-powered Web UI lets you control processing remotely:

  • Update vocoder configuration live
  • Start/stop audio processing
  • Choose visualization modes
  • Adjust advanced parameters from any device on your network

To access the Web UI, navigate to: 

http://localhost:5000

10. Customizing Themes

The app supports theming via themes/web_ui.html and themes/custom_theme.css. You can customize:

  • Font and color scheme (e.g. ParrotOS neon green/cyan)
  • Button styling and layout
  • Background images and assets

11. Assets Guide

Required files in assets/:

  • logo_screen.png: Splash screen image
  • credits_screen.png: Credits/about screen
  • exit_screen.png: Exit/closing screen
  • static_visualizer.png: Visualizer background
  • shape_predictor_68_face_landmarks.dat: Dlib face landmark model (auto-downloaded)
  • default_model.obj: 3D placeholder model
  • waifu_mascot.obj: Custom 3D mascot model

12. Troubleshooting

  • Missing shape_predictor_68_face_landmarks.dat? App will attempt to auto-download.
  • No trained_voice_model.pt? Voice cloning will be disabled; use placeholders for testing.
  • Audio device errors? Check your microphone and output device settings.
  • Dependency errors? Verify Python version and install requirements.txt again.
